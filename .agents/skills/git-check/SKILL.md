# Git 凭证检查技能

**用途：** 推送到GitHub之前，检查凭证状态并测试连接，避免推送失败。

## 触发方式：
`/git-check`

## 执行步骤：

1. **检查git配置**
   - 检查 remote 地址配置
   - 检查 git 凭证存储配置
   - 检查当前用户信息

2. **测试网络连接**
   - 测试 GitHub 连通性
   - 验证凭证有效性
   - 检查权限

3. **干运行测试推送**
   - 执行 `git push --dry-run`
   - 捕获认证错误
   - 给出修复建议

4. **结果报告**
   - 如果一切正常，可以安全推送
   - 如果有问题，分步指导修复认证问题

## 错误处理指导：

如果发现认证失败：
1. 提示用户检查 Personal Access Token 是否过期
2. 提示用户检查 token 是否有正确权限（repo scope）
3. 指导更新本地 git 凭证存储
4. 测试连接后再重试推送
