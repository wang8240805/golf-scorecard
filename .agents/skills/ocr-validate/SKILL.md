# OCR 验证技能

**用途：** 每次修改高尔夫记分卡OCR识别代码后，自动运行完整验证检查。

## 触发方式：
`/ocr-validate`

## 执行步骤：

1. **读取OCR代码** - 读取 `cloudfunctions/ocr-scorecard/index.js` 完整代码

2. **语法检查**
   - 检查是否有语法错误（括号匹配、逗号等）
   - 检查是否有重复变量声明
   - 检查函数是否完整定义

3. **验证识别逻辑完整性**
   - ✅ 确认 `parseTableStructured()` 函数存在并输出完整调试日志
   - ✅ 确认每一步识别都有日志输出（步骤1-步骤5）
   - ✅ 确认每个文字块都输出坐标信息
   - ✅ 确认自动检测第二行洞号逻辑存在（当提取少于18时）
   - ✅ 确认只提取关键字**下方**块，过滤上方无关码数
   - ✅ 确认支持中文表头（洞号/标准杆/杆）
   - ✅ 确认任何数字串都逐字符拆分，处理OCR合并
   - ✅ 确认验证不通过时调用 `fixWithAI()` AI修复
   - ✅ 确认 **不补全不推断，只返回真实识别结果**

4. **验证高尔夫规则检查**
   - ✅ PAR值只能是 3、4、5
   - ✅ 每9洞总标准杆固定为 36
   - ✅ 18洞总标准杆固定为 72
   - ✅ 校验不通过走AI修复流程

5. **输出验证报告**
   - 列出所有检查项
   - 标记通过/不通过
   - 如果有问题给出具体修复建议

## 输出格式：

```
## OCR 验证报告

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 语法正确 | ✅ 通过 | - |
| 调试日志完整 | ✅ 通过 | 每步都有详细日志 |
| ... | ... | ... |

**结论：** [通过/需要修复]
```
