# PR 描述生成技能

**用途：** 自动分析代码变更，生成规范的 Pull Request 描述。

## 触发方式：
`/pr-description [base-branch]`

## 执行步骤：

1. **获取变更**
   - 运行 `git diff [base-branch]...HEAD` 获取所有变更
   - 统计文件和代码行数变化

2. **分类变更类型**
   - `feat`: 新功能
   - `fix`: Bug修复
   - `refactor`: 重构
   - `docs`: 文档更新
   - `perf`: 性能优化
   - `test`: 测试相关

3. **生成描述**
   - 一句话概括变更
   - 列出主要变更点
   - 关联相关问题（如果有）
   - 添加测试清单

4. **输出格式**
   - 遵循 conventional commits 格式
   - 生成可直接复制的 Markdown

## 输出模板：

```markdown
## 摘要

[一句话描述这个PR解决了什么问题]

## 变更类型

- [ ] 新功能
- [ ] Bug修复
- [ ] 代码重构
- [ ] 文档更新
- [ ] 性能优化
- [ ] 测试

## 主要变更

- [变更点1]
- [变更点2]
- [变更点3]

## 测试计划

- [ ] 现有测试通过
- [ ] 新增测试覆盖新逻辑
- [ ] 手动测试验证功能正常

## 相关问题

Fixes #XXX
```
